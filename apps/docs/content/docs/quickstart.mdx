# Quickstart

This is the fastest path to a working self-hosted setup.

## 1. Install the package

```bash
npm i opendetail
```

## 2. Scaffold the default setup

```bash
bunx opendetail setup --with-media
```

That command can:

- write `opendetail.toml`
- scaffold `src/app/api/opendetail/route.ts`
- build `.opendetail/index.json`

## 3. Add your runtime key

Set `OPENAI_API_KEY` in the app runtime environment before you send requests to the route.

## 4. Check the generated config

The default shape looks like this:

```toml
version = 1
include = ["content/**/*.{md,mdx}"]
exclude = []
base_path = "/docs"

[media]
include = ["content/**/*.{png,jpg,jpeg,webp,avif,gif,svg}"]
exclude = []
base_path = "/content-media"
```

## 5. Rebuild when docs change

```bash
bunx opendetail build
```

## 6. Ask a question

With the default Next.js route in place, post JSON like this:

```json
{ "question": "How do I install this?" }
```

The route streams NDJSON events in this order:

- `meta`
- `sources`
- `images`
- `delta`
- `done`
- `error`

## Hosted instead

If you want the hosted integration shape without generating a route:

```bash
bunx opendetail setup --integration hosted
```

That still writes config and builds the local index, but it skips route scaffolding on purpose.
