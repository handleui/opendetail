# Visual References

`opendetail` can carry image references from your docs into the answer payload. That lets a UI show a visual next to the response instead of only rendering text and citations.

![Thread shell reference](/opendetail-test/thread-shell.svg "Thread Shell Reference")

## What the first visual demonstrates

The shell reference shows a narrow reading column with a fixed composer anchored near the bottom of the viewport. It is useful when a product wants the input to stay reachable while the answer stream grows above it.

![Streaming response reference](/opendetail-test/stream-reference.svg "Streaming Response Reference")

## What the second visual demonstrates

The streaming response reference emphasizes the assistant answer card, source metadata, and the way an image can arrive before the full body copy has finished streaming. That makes it a good test case for the live UI surface in the website app.
