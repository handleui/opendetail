Assistant UI documentation is **reference**: **hooks** (`useOpenDetail`, `createOpenDetailClient`) and **primitives** (layout, thread, composer, messages) with live previews.

**Styling** — systems, themes, and what “primitives” means in the architecture: [**Styling**](/docs/design-system).

In React, a **hook** is a function that follows the `use*` convention and lets you **subscribe to React state and lifecycle** from function components. **`useOpenDetail`** owns **thread state**, **streaming NDJSON**, **persistence**, and **submit / stop** controls. When you **cannot** use a hook—vanilla JS or a store you manage yourself—use **`createOpenDetailClient`**, the same **`POST`** / **`application/x-ndjson`** contract without React bindings.

| API | Use when |
| --- | --- |
| [`useOpenDetail`](/ui/hooks/use-opendetail) | React state: thread, question, `submit` / `stop`, persistence, streaming updates bound to UI. |
| [`createOpenDetailClient`](/ui/hooks/create-open-detail-client) | Same stream **without** the hook (vanilla JS, custom state, non-React surfaces). |

**Primitives** ([**Styling**](/docs/design-system#primitives)) are the styled building blocks above these clients; the full **`opendetail-react`** API is in [**React**](/docs/react).
