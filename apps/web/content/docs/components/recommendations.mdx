---
title: Recommendations
description: AssistantSuggestions and AssistantSuggestion render tappable follow-up prompts—chips you can place under the thread or near the input.
---

Use **`AssistantSuggestions`** as a row or wrap container and **`AssistantSuggestion`** for each chip. Both are client components; suggestions are plain buttons with optional **`disabled`** and **`onClick`**.

Styling uses `opendetail-assistant-suggestions` and `opendetail-assistant-suggestion` (plus `opendetail-pressable`) from the OpenDetail base stylesheet.

## Props

### `AssistantSuggestions`

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | — | Typically several `AssistantSuggestion` elements. |
| `className` | `string` | — | Optional class on the list wrapper. |

### `AssistantSuggestion`

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | — | Label for the chip. |
| `onClick` | `() => void` | — | Called when the suggestion is activated (keyboard or pointer). |
| `disabled` | `boolean` | `false` | Disables interaction and dims the chip. |
| `className` | `string` | — | Optional class on the `button`. |

## Composition

Wrap multiple `AssistantSuggestion` nodes inside `AssistantSuggestions`. In the [Shell](/components/shell) preview they sit **inside the thread** so they scroll with the conversation; you can also render them directly under **`ConversationLayout`**’s input if you prefer them fixed.
