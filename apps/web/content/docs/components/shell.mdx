## `AssistantShell` (layout primitive)

**AssistantShell** is the vertical layout that pairs the **scrollable conversation** with the **fixed prompt**. It does not include the sidebar header (copy, new chat, collapse) or model streaming—you compose those yourself or use **`AssistantSidebarShell`**.

Regions use semantic hooks (`opendetail-shell`, `opendetail-shell__thread`, `opendetail-shell__input`) so you can theme or measure them consistently.

### Composition

The `thread` slot usually contains **`AssistantThread`**, **`AssistantUserMessage`** / **`AssistantResponse`**, and optionally **`AssistantSuggestions`** (see [Input](/components/input) and [Recommendations](/components/recommendations)).

### Props (`AssistantShell`)

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `input` | `ReactNode` | — | **Required.** Almost always an `AssistantInput`. |
| `thread` | `ReactNode` | — | Scrollable conversation. Prefer this when you have a structured thread. |
| `children` | `ReactNode` | — | If `thread` is omitted, `children` render inside the thread region. |
| `className` | `string` | — | Optional class on the root `section`. |

### Usage

```tsx
import {
  AssistantInput,
  AssistantShell,
  AssistantThread,
  AssistantUserMessage,
} from "opendetail-react";

<AssistantShell
  input={<AssistantInput name="q" size="shell" onSubmit={handleSubmit} />}
  thread={
    <AssistantThread>
      <AssistantUserMessage>Hello</AssistantUserMessage>
    </AssistantThread>
  }
/>;
```

## Site assistant (`AssistantSidebar` / `AssistantSidebarShell`)

What you see in the **live preview** below is the same stack as this docs app: **`AssistantSidebarShell`** → **`useOpenDetail`** → **`POST /api/opendetail`**, with **`renderNextSourceLink`** and a Fumadocs-style **`resolveSourceTarget`** so citations resolve safely.

That layer adds the **header** (title, **copy thread**, **new chat**, **collapse**), the **resizable** panel, default **prompt suggestions** when the thread is empty, and the wired **`AssistantInput`** / thread rendering—you do not reimplement those for a standard app shell.

### `AssistantSidebar` — `embedded`

For bounded in-page demos (like this page), pass **`embedded`** on **`AssistantSidebar`** (or through **`AssistantSidebarShell`**): one assistant column, no separate main `children` slot, fixed height, an **Open assistant** control if the user collapses the panel, and a **400px** max width so the prompt matches the column.

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `embedded` | `boolean` | `false` | Single-column preview: same sidebar UI without a main content sibling. |

Other sidebar props (`headerTitle`, `messages`, `onSubmitQuestion`, `promptSuggestions`, `sidebarResizeEnabled`, etc.) are unchanged—see **`AssistantSidebar`** / **`AssistantSidebarShell`** in the main [React](/docs/react) docs and [Packages & composition](/docs/packages).
