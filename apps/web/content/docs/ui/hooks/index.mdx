title: Hooks overview
description: Headless browser APIs in opendetail-react — when to use the hook vs the HTTP client.
---

`opendetail-react` exposes two main ways to drive the same NDJSON pipeline in the browser. This page is a **quick orientation**; the sidebar links to dedicated pages for each API, and [**React**](/docs/react) is the full reference (options, stream events, types).

| API | Use when |
| --- | --- |
| [`useOpenDetail`](/ui/hooks/use-opendetail) | You want React state: thread, question, `submit` / `stop`, persistence, and streaming updates bound to UI. |
| [`createOpenDetailClient`](/ui/hooks/create-open-detail-client) | You need streaming without the hook (vanilla JS, custom state, or non-React embeds). |

Both talk to the same **`POST`** endpoint and event types. [**Primitives**](/ui/primitives) cover the styled components that typically sit *above* these clients—not a third transport, just layout and presentation.
