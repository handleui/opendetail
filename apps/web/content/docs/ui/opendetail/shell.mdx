## `ConversationLayout` (shell variant)

**`ConversationLayout`** with **`variant="shell"`** is the vertical layout that pairs the **scrollable conversation** with the **fixed prompt**. It does not include the sidebar header (copy, new chat, collapse) or model streaming—you compose those yourself or use **`AssistantSidebar`** with **`connection`**.

Regions use semantic hooks (`opendetail-shell`, `opendetail-shell__thread`, `opendetail-shell__input`) so you can theme or measure them consistently.

### Composition

The `thread` slot usually contains **`Thread`**, **`UserMessage`** / **`AssistantMessage`**, and optionally **`AssistantSuggestions`** (see [Composer](/ui/opendetail/composer) and [Recommendations](/ui/opendetail/recommendations)).

### Props (`ConversationLayout`, shell)

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `variant` | `"shell"` | — | **Required** for this layout. |
| `input` | `ReactNode` | — | **Required.** Almost always a **`Composer`**. |
| `thread` | `ReactNode` | — | Scrollable conversation. Prefer this when you have a structured thread. |
| `children` | `ReactNode` | — | If `thread` is omitted, `children` render inside the thread region. |
| `className` | `string` | — | Optional class on the root `section`. |

### Usage

```tsx
import {
  Composer,
  ConversationLayout,
  Thread,
  UserMessage,
} from "opendetail-react";

<ConversationLayout
  input={<Composer name="q" size="shell" onSubmit={handleSubmit} />}
  thread={
    <Thread>
      <UserMessage>Hello</UserMessage>
    </Thread>
  }
  variant="shell"
/>;
```

## Site assistant (`AssistantSidebar`)

What you see in the **live preview** below is the same stack as this docs app: **`AssistantSidebar`** with **`connection`** → **`useOpenDetail`** → **`POST /api/opendetail`**, with **`renderNextSourceLink`** and a Fumadocs-style **`resolveSourceTarget`** so citations resolve safely.

This **Shell** preview uses **`embedded` + `embeddedLayout="compact"`** (rounded **400px** column), **`embeddedHideCollapse`** (no collapse control—focus stays on thread + input), plus **copy**, **new session**, resize, suggestions, and **`Composer`** filling the column width.

On the [**Sidebar**](/ui/opendetail/sidebar) page, the preview uses **`embeddedLayout="dock"`** instead: the panel sits flush on the **left** of the white preview with a **left stroke only** (no outer rounded chrome), and **collapse** + **Open assistant** behave like the full app.

### `AssistantSidebar` — `embedded`

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `embedded` | `boolean` | `false` | Single-column preview: same sidebar UI without a main content sibling. |
| `embeddedLayout` | `"compact" \| "dock"` | `"compact"` | **`compact`** — 400px rounded box; **`dock`** — left-aligned column, left border only (see [Sidebar](/ui/opendetail/sidebar)). |
| `embeddedHideCollapse` | `boolean` | `false` | Hide the collapse icon (used on this page’s preview). |

Other sidebar props (`headerTitle`, `messages`, `onSubmitQuestion`, `promptSuggestions`, `sidebarResizeEnabled`, etc.) are unchanged—see **`AssistantSidebar`** in the main [React](/docs/react) docs and [Packages & composition](/docs/packages).
