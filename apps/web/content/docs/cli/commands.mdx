The **`opendetail`** package exposes a small CLI: **`build`** (index your docs), **`setup`** (scaffold config and route), and **`doctor`** (verify the setup). Run it via **`npx`**, **`bunx`**, or **`pnpm dlx`** so the version matches your `package.json`.

## Commands

| Command | Purpose |
| --- | --- |
| [`opendetail build`](/docs/cli/build) | Write `.opendetail/index.json` from your config and content globs. |
| [`opendetail setup`](/docs/cli/setup) | Create `opendetail.toml`, optional route, optional first build. |
| [`opendetail doctor`](/docs/cli/doctor) | Check files and env for the selected integration mode. |

Each command has its own page with full flags and copy-paste examples.

## One-liner usage

<CodeBlock lang="bash">{`opendetail build [--config <path>] [--output <path>] [--cwd <path>]
opendetail setup [--integration <self-hosted|hosted>] [--config <path>] [--route <path>] [--base-path <url>] [--include <glob>] [--media-include <glob>] [--with-media] [--skip-build] [--force] [--interactive] [--no-interactive] [--cwd <path>]
opendetail doctor [--integration <self-hosted|hosted>] [--config <path>] [--route <path>] [--cwd <path>]
`}</CodeBlock>

## Suggested order

1. [`setup`](/docs/cli/setup) once per repo (or when you change integration mode).
2. [`build`](/docs/cli/build) whenever docs or media globs change.
3. [`doctor`](/docs/cli/doctor) when something fails or after upgrading packages.
