`opendetail setup` writes the **standard project shape** for either **self-hosted** (local Next route + OpenAI) or **hosted** (client talks to your deployed endpoint).

## Usage

<CodeBlock lang="bash">{`opendetail setup [--integration <self-hosted|hosted>] [--config <path>] [--route <path>] [--base-path <url>] [--include <glob>] [--media-include <glob>] [--with-media] [--skip-build] [--force] [--interactive] [--no-interactive] [--cwd <path>]
`}</CodeBlock>

## What it can write

- **`opendetail.toml`**
- In **self-hosted** mode, a route at `src/app/api/opendetail/route.ts` by default (override with `--route`)
- **`.opendetail/index.json`** unless you pass `--skip-build`

## Important flags

| Flag | Meaning |
| --- | --- |
| `--integration` | `self-hosted` or `hosted`. |
| `--config` | Custom config path. |
| `--route` | Custom route file path (self-hosted). |
| `--base-path` | Public source URL base (default `/docs`). |
| `--include` | Docs glob (default matches typical `content/**/*.{md,mdx}`). |
| `--media-include` | Media glob when media is enabled. |
| `--with-media` | Include `[media]` in config. |
| `--skip-build` | Skip the initial index build. |
| `--force` | Overwrite generated files. |
| `--interactive` / `--no-interactive` | Control the setup wizard when stdin is a TTY. |
| `--cwd` | Run against another working directory. |
