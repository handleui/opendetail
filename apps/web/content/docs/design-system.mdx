OpenDetail splits **systems** (full UI identity: structure, semantics, layout) from **themes** (palette layers on top of one system). **Primitives** are the documented React components that implement the system. The default **OpenDetail** system and **Midnight** / **Signal** themes ship with `opendetail-react` styles.

**Headless API (same for every system):** [**React**](/docs/react) — `useOpenDetail`, `createOpenDetailClient`. **Live component reference** (props and previews): [**Assistant UI**](/ui).

## Systems

A **system** is a full **UI identity** for the assistant: semantic class contract, layout and density, motion, and component anatomy (thread, composer, messages, sources). The default **OpenDetail** system ships in `opendetail-react` under `src/styles/systems/opendetail/` (aggregated by **`opendetail-base.css`**).

Add a new system when you need **different chrome or structure**—not just different colors—while keeping the same headless transport (`useOpenDetail`, `createOpenDetailClient`) and message shapes.

### Systems vs themes

- **Themes** are palette layers **inside** one system: same layout and components, different tokens. See [Themes](#themes).
- **Systems** stay isolated when a product line needs different chrome but the same streaming protocol.
- In the repo: new colors → `themes/`; new chrome → `systems/`.

### Import order (system CSS)

1. Import the **base** system stylesheet (`opendetail-base.css` or `systems/opendetail/index.css`).
2. Optionally layer a [theme](#themes) after base (`opendetail-midnight.css`, `opendetail-signal.css`, or a custom palette file).

Root elements use **`data-od-system="opendetail"`** where layout marks the design system for theming.

## Themes

A **theme** is a **palette** (and optional minor token tweaks) **inside** one [system](#systems): background, foreground, accent, borders. Layout and React [primitives](#primitives) stay the same; only paint changes.

Themes are **palette layers** on top of **`opendetail-base.css`**: same layout classes and components; only tokens change. Import **base first**, then **one theme file**.

### Midnight

**Midnight** uses a **cooler dark** palette: semantic color tokens (background, foreground, accent) shift while structure stays the same.

```css
@import "opendetail-base.css";
@import "opendetail-midnight.css";
```

Package paths follow your bundler’s resolution for `opendetail-react` styles (see the [style README](https://github.com/handleui/opendetail/blob/main/packages/opendetail-react/src/styles/README.md)).

### Signal

**Signal** is a **warmer** accent / palette variant on the same base contract—layout and components are unchanged.

```css
@import "opendetail-base.css";
@import "opendetail-signal.css";
```

### Adding your own

Custom themes belong next to the base import: override semantic CSS variables scoped to **`data-od-system="opendetail"`** (or your system) without forking React primitives.

## Primitives

**Primitives** are the **documented React exports** that implement the OpenDetail **system**: layout regions, the thread and composer, messages, sources, and status—styled by **`opendetail-base.css`** and optional [themes](#themes). They are **not** the headless streaming layer; that lives in **`useOpenDetail`** and **`createOpenDetailClient`** ([**useOpenDetail**](/ui/hooks/use-opendetail), [**createOpenDetailClient**](/ui/hooks/create-open-detail-client), full reference [**React**](/docs/react)).

### What ships

1. **Layout** — **`ConversationLayout`** (thread + composer regions); **`AssistantSidebar`** and **`AssistantModal`** compose it for a full column or modal assistant.
2. **Conversation UI** — **`Thread`**, **`Composer`**, **`UserMessage`**, **`AssistantMessage`**, **`AssistantSources`**, suggestions, **`AssistantStatus`**, and source link helpers.
3. **Styles** — Structural tokens and component classes from the [system](#systems) stylesheet; palette from themes.

**Recipes** (`AssistantSidebar`, `AssistantModal`) bundle primitives for a drop-in experience; you can also compose primitives inside your own shell.

Browse **Primitives** in the [Assistant UI](/ui) sidebar for live previews and props—[**Shell**](/ui/opendetail/shell) is a compact entry point.
