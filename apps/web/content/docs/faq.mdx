# FAQ

## Does OpenDetail ship a chat UI?

No. OpenDetail is the engine and integration layer. The current package focuses on the build step, local index, runtime, and thin integration helpers.

## Is it built on OpenAI?

Yes. OpenDetail uses the OpenAI Responses API underneath.

## Do I need a vector database?

No. The default path builds a local JSON index and searches it locally.

## What content gets indexed?

Whatever matches the `include` globs in `opendetail.toml`, minus anything in `exclude`.

## Does MDX still work if the file uses components?

Yes, but the important part is the text and headings. Retrieval is about the content, not the rendered component tree.

## Can it answer from outside my docs?

Not by default. The local path is grounded in matched docs only. If you configure `remote_resources`, it can also cite those remote tool results.

## What happens when the docs are not enough?

It answers plainly that the topic is not documented yet or cannot be confirmed from the configured docs.

## Where does the data live?

The built index lives in your app at `.opendetail/index.json`.

## Is the hosted backend available yet?

No. The hosted integration is documented and scaffolded in this phase, but the OpenDetail-managed backend is not shipped yet.

## What frameworks are supported today?

Today the package family is split into `opendetail`, `opendetail-react`, `opendetail-next`, and `opendetail-fumadocs`.

## Does it replace docs?

No. OpenDetail works because the docs already exist. It makes that material easier to ask, retrieve, and cite inside an app.
