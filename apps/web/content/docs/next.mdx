title: Next.js
description: Next.js App Router helpers for the OpenDetail backend contract and safe source links.

Use **`opendetail-next`** when your app backend is Next.js and you want the thinnest possible route adapter on top of OpenDetail. It is a good fit for the **Fastest** path, and it also works for **Branded** or **Headless** when Next.js is simply the server runtime you already have.

## Route handler

The scaffolded route is just a wrapper around the backend contract:

<CodeBlock lang="ts">{`import { createNextRoute } from "opendetail-next";

export const { POST, runtime } = createNextRoute();`}</CodeBlock>

Or wire the handler yourself:

<CodeBlock lang="ts">{`import { createNextRouteHandler } from "opendetail-next";

export const runtime = "nodejs";
export const POST = createNextRouteHandler();`}</CodeBlock>

## What the route expects

The route accepts `POST` requests with JSON like this:

```json
{ "question": "How do I configure public_path?" }
```

If the body is malformed, the handler returns JSON error data. If the answer can stream, the response uses NDJSON and emits the same event types as the rest of OpenDetail: `meta`, `sources`, `images`, `delta`, `done`, and `error`.

That contract is what custom UIs and agents should rely on. The route adapter does not change the meaning of the stream; it only exposes it through Next.js.

## Requirements

- Node.js 18+
- `runtime = "nodejs"` for the default adapter
- `OPENAI_API_KEY` available where the route runs
- `.opendetail/index.json` present after a build

`createNextRouteHandler` accepts the same core options as `createOpenDetail`, so you can override `cwd`, `indexPath`, or client injection when needed.

## Source links

For docs sites, `renderNextSourceLink` renders local citations with `next/link` and keeps external URLs as safe anchors.

If your docs live at known page URLs, pair it with the helper from [`opendetail-fumadocs`](/docs/fumadocs) so source targets stay on allowed paths instead of inventing new links.
