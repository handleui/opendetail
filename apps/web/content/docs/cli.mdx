The CLI ships in the [`opendetail`](https://www.npmjs.com/package/opendetail) package. Prefer **`npx`**, **`bunx`**, or **`pnpm dlx`** so you always run the version pinned by your project.

The **`opendetail`** package exposes **`build`** (index your docs), **`setup`** (scaffold config and route), and **`doctor`** (verify the setup).

## Install and first run

<CopyCommand bun="bunx opendetail --version" npm="npx opendetail --version" />

If that prints a version, the executable resolves correctly from your package manager.

Running the CLI without a subcommand prints supported commands and example invocations (same sections below).

### Typical flows

- **Greenfield app:** [`opendetail setup`](#setup) to write `opendetail.toml`, scaffold the default route (self-hosted), and optionally run an initial index build.
- **Docs changed:** [`opendetail build`](#build) to refresh `.opendetail/index.json`.
- **Something looks wrong:** [`opendetail doctor`](#doctor) to verify config, index, route, and environment expectations.

## Command overview

| Command | Purpose |
| --- | --- |
| `opendetail build` | Write `.opendetail/index.json` from your config and content globs. |
| `opendetail setup` | Create `opendetail.toml`, optional route, optional first build. |
| `opendetail doctor` | Check files and env for the selected integration mode. |

### One-liner usage

<CodeBlock lang="bash">{`opendetail build [--config <path>] [--output <path>] [--cwd <path>]
opendetail setup [--integration <self-hosted|hosted>] [--config <path>] [--route <path>] [--base-path <url>] [--include <glob>] [--media-include <glob>] [--with-media] [--skip-build] [--force] [--interactive] [--no-interactive] [--cwd <path>]
opendetail doctor [--integration <self-hosted|hosted>] [--config <path>] [--route <path>] [--cwd <path>]
`}</CodeBlock>

### Suggested order

1. [`setup`](#setup) once per repo (or when you change integration mode).
2. [`build`](#build) whenever docs or media globs change.
3. [`doctor`](#doctor) when something fails or after upgrading packages.

## build

`opendetail build` is the **index build** step: it resolves paths from your config, walks the include globs, and writes **`.opendetail/index.json`** (or `--output`).

### Usage

<CodeBlock lang="bash">{`opendetail build [--config <path>] [--output <path>] [--cwd <path>]
`}</CodeBlock>

### Flags

| Flag | Meaning |
| --- | --- |
| `--config` | Path to `opendetail.toml` (default: `./opendetail.toml`). |
| `--output` | Where to write the index JSON (default: `.opendetail/index.json` relative to cwd). |
| `--cwd` | Working directory for resolving paths. |

### Examples

<CopyCommand bun="bunx opendetail build" npm="npx opendetail build" />

<CopyCommand bun="bunx opendetail build --config ./config/opendetail.toml" npm="npx opendetail build --config ./config/opendetail.toml" />

## setup

`opendetail setup` writes the **standard project shape** for either **self-hosted** (local Next route + OpenAI) or **hosted** (client talks to your deployed endpoint).

### Usage

<CodeBlock lang="bash">{`opendetail setup [--integration <self-hosted|hosted>] [--config <path>] [--route <path>] [--base-path <url>] [--include <glob>] [--media-include <glob>] [--with-media] [--skip-build] [--force] [--interactive] [--no-interactive] [--cwd <path>]
`}</CodeBlock>

### What it can write

- **`opendetail.toml`**
- In **self-hosted** mode, a route at `src/app/api/opendetail/route.ts` by default (override with `--route`)
- **`.opendetail/index.json`** unless you pass `--skip-build`

### Important flags

| Flag | Meaning |
| --- | --- |
| `--integration` | `self-hosted` or `hosted`. |
| `--config` | Custom config path. |
| `--route` | Custom route file path (self-hosted). |
| `--base-path` | Public source URL base (default `/docs`). |
| `--include` | Docs glob (default matches typical `content/**/*.{md,mdx}`). |
| `--media-include` | Media glob when media is enabled. |
| `--with-media` | Include `[media]` in config. |
| `--skip-build` | Skip the initial index build. |
| `--force` | Overwrite generated files. |
| `--interactive` / `--no-interactive` | Control the setup wizard when stdin is a TTY. |
| `--cwd` | Run against another working directory. |

### Examples

<CopyCommand bun="bunx opendetail setup --with-media" npm="npx opendetail setup --with-media" />

<CopyCommand bun="bunx opendetail setup --integration hosted" npm="npx opendetail setup --integration hosted" />

## doctor

`opendetail doctor` runs a **checklist** and prints âś“ or âś— for each item. It exits with an error if anything required is missing.

### Usage

<CodeBlock lang="bash">{`opendetail doctor [--integration <self-hosted|hosted>] [--config <path>] [--route <path>] [--cwd <path>]
`}</CodeBlock>

### Self-hosted expectations

- **Config:** `opendetail.toml` (or `--config`) exists.
- **Index:** `.opendetail/index.json` (or configured output) exists.
- **Route:** the API route file exists at the configured path.
- **Env:** `OPENAI_API_KEY` is set where the route runs.

### Hosted expectations

- **Config** and **index** as above.
- **Env:** `OPENDETAIL_ENDPOINT` points at your hosted OpenDetail HTTP API.
